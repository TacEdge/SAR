Nominal is green by default. Active operations, go states and a successful sync reuse Forest rather than a second green, so the palette stays tight. The signal layer earns its place only where the earth tones cannot mean urgency.

Domain accent

Land Marine The one config that recolours the chrome. The M1 land / marine flag drives the whole product; the accent follows it, forest for land, slate-blue for marine. Toggle it in the header.

The rule, applied identically on all nine screens. Every screen defines --accent:var(--forest) with --accent-tint:var(--forest-tint) at :root, and a single html[data-domain="marine"] override swapping in --marine / --info-bg. The data-domain attribute is set by the workspace shell (and by each screen's own land / marine control where present), so the accent is deliberate domain theming, never a per-screen choice. Any accented affordance uses var(--accent), never a hard-coded forest or marine.

Typography

IBM Plex Sans and IBM Plex Mono, one stack across desktop and field. No display face: a loud headline would fight the calm this product needs, so personality lives in the mono data language instead. Settling on a single family here retires the desktop-coherence debt the Geotech PM surface carried.

The mono data language — the signature

Every fact-of-record is set in IBM Plex Mono: coordinates, grid references, times, callsigns, channels, reference numbers, deployed hours. It reads like an instrument, keeps columns of data scannable, and marks clearly what is a recorded fact versus prose.

04

Two surfaces, one system

The product spans an ops-room desktop and a hillside phone. They share every token; they differ only in density and target size. Deciding both now, from one system, is what keeps them coherent.

Coordinator desktop

Dense, multi-panel, indoor

  • Base text 15px, compact density, minimum target 32px
  • Multi-panel: the COP is a map with docked log and task panels
  • Mouse and keyboard; keyboard shortcuts for the common path
  • Mono data in tight columns; more information per screen
M1 · M2 · M3 · M5 · M9
Field mobile

Sparse, large-target, outdoor

  • Base text 16px, generous spacing, minimum target 48px
  • Single column, thumb-reachable, sunlight-legible contrast
  • Offline state always visible; capture never blocks on signal
  • Night mode is a first-class variant, not an afterthought
M4 · pushed tasks & briefings

07

Night mode — field-first

SAR runs at night. The field surface needs a dark variant that preserves the same semantics at lower luminance, so a screen does not blind a searcher or wreck night vision. This is a required variant, defined here as direction and confirmed in the prototype.

Field night palette

Ink inverts to a warm off-black; Forest lifts to a legible foliage green; the signal layer keeps its meaning but drops in intensity. Text targets AA at reduced brightness.

Token reference

The custom properties the M1 and M2 screens import. This file is the source of truth; copy the block into each screen's :root so nothing drifts.
